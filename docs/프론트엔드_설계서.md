# Frontend 설계서 (Vue3)

## 1. 기술 스택
- Vue 3 (Composition API), Vite
- Vue Router, Pinia
- TailwindCSS
- Axios(현재 mock API 호출 구조)

## 2. 디렉토리 구조
- `src/app/router`: 라우트/가드
- `src/app/store`: 전역 상태(auth)
- `src/app/layouts`: blank/user/admin 레이아웃
- `src/pages/admin`: 행사/배치/관리
- `src/pages/user`: 조회/신청/마이
- `src/services/api/modules`: auth/events/admin/apply
- `src/services/api/mock-db.js`: 로컬 스토리지 mock DB

## 3. 라우팅 설계
### 공통
- `/` 로그인
- `/register` 회원가입

### 사용자
- `/join-list` 행사 조회
- `/apply` 배치 코드 조회/배치도 조회
- `/mypage` 마이페이지

### 관리자
- `/index/admin/event-register` 행사 등록
- `/index/admin/event-list` 행사 목록
- `/index/admin/event-layout?eventId=...` 배치도 편집
- `/index/admin/event-participants` 참여자 조회(placeholder)
- `/index/admin/excel-download` 엑셀 다운로드(placeholder)

### 권한 가드
- `guestOnly`: 로그인 시 role 기반 리다이렉트
- `requiresAuth`: 비로그인 접근 차단
- `role=ADMIN`: 일반 사용자 접근 차단

## 4. 데이터 모델 (프론트 관점)
```ts
type Event = {
  id: number
  title: string
  startDate: string
  endDate: string
  period: string
  manager: string
  posterImages: string[]
  content: string
  description: string
  layoutCode: string | null
  layout: {
    rows: number
    cols: number
    cells: LayoutCell[]
  }
}

type LayoutCell = {
  id: number
  label: string
  name: string
  content: string
}
```

## 5. 핵심 로직
### 5.1 행사 생성
- `EventRegisterPage.vue`
- 필수값 검증 후 `registerEvent(payload)` 호출
- 성공 시 `event-layout?eventId={id}` 이동

### 5.2 배치 편집 및 코드 발급
- `LayoutEditorPage.vue`
- `fetchLayout(eventId)`로 배치 로드
- `vue-draggable-plus`로 셀 순서 변경
- `saveLayout(eventId, { rows, cols, cells })` 호출
- 저장 시 코드 미발급 상태면 4자리 코드 생성

### 5.3 사용자 코드 조회
- `ApplyPage.vue`
- 코드 입력 모달에서 `4자리 숫자` 검증
- `fetchApplyLayoutByCode(code)` 성공 시 배치도 표시
- 셀 클릭 시 셀 상세 모달 노출

## 6. API 계약 (현재 mock)
### events
- `fetchEventList()`
- `fetchEventDetail(eventId)`
- `registerEvent(payload)`

### admin
- `fetchLayout(eventId)`
- `saveLayout(eventId, payload)`

### apply
- `fetchApplyLayoutByCode(layoutCode)`

### auth
- `signIn({userId,password})`
- 현재 허용 계정: `admin / 1234`
