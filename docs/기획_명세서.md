# Layout Platform 기획 명세서

## 1. 목적
행사 운영자가 행사/배치도를 생성하고, 사용자가 4자리 배치 코드로 배치도를 조회해 셀 정보를 확인할 수 있는 웹 서비스를 제공한다.

## 2. 역할
- 관리자(ADMIN): 행사 생성, 배치도 생성/편집, 배치 코드 발급
- 사용자(USER): 배치 코드 입력 조회, 셀 정보 확인

## 3. 핵심 시나리오
### 3.1 관리자
1. 행사 등록
- 입력: 행사명, 행사기간(start/end), 담당자, 행사 포스터(n개), 행사 내용
- 결과: 행사 생성, 배치도 편집 화면으로 이동

2. 배치도 생성/편집
- 셀 클릭 시 모달에서 이름/내용 입력 또는 초기화
- 셀 드래그앤드롭으로 순서 이동
- 배치 저장 시 이벤트에 4자리 배치 코드 발급(최초 1회)

3. 행사 목록
- 이벤트별 코드 발급 상태 확인
- 특정 행사 배치 편집 화면으로 이동

### 3.2 사용자
1. 배치 코드 조회
- 진입 시 코드 입력 모달 노출
- 4자리 코드 입력 후 일치하면 배치도/행사 정보 표시

2. 셀 정보 확인
- 셀 클릭 시 이름/내용 모달 표시

## 4. 화면 요구사항
- 사용자 레이아웃: 헤더/푸터 고정, 가운데 콘텐츠 스크롤
- 웹 반응형: 데스크탑 중심폭(70vw), 소형 화면에서 full width
- 관리자 레이아웃: 사이드바(검정), 콘텐츠(흰색)

## 5. 상태/에러 정책
- 로그인 실패: 에러 토스트
- 코드 조회 실패(미일치): 에러 토스트
- 잘못된 코드 형식: 4자리 숫자 검증
- 행사/배치 없음: 안내 메시지

## 6. 수용 기준 (Acceptance Criteria)
- [AC-01] 행사 등록 필수값 미입력 시 저장 불가
- [AC-02] 배치 저장 시 코드가 없으면 4자리 코드 발급
- [AC-03] 이미 코드가 있는 배치는 저장해도 코드 유지
- [AC-04] 사용자 코드가 일치할 때만 배치도 렌더링
- [AC-05] 셀 클릭 시 이름/내용 상세 확인 가능
- [AC-06] 헤더/푸터는 항상 보이고, 콘텐츠만 스크롤

## 7. 범위 외(현재)
- 실제 파일 업로드 저장(S3/스토리지)
- 실제 사용자 신청 확정/취소 백엔드 처리
- 관리자 참여자 통계/엑셀 실데이터
